# 变更：修复学习笔记页面乱码显示

## 为什么
学习笔记模块当前统一使用 UTF-8 读取 `study/` 目录下的文件内容。现有部分历史笔记文件实际采用 GBK/GB18030 等中文编码保存，导致页面展示出现明显乱码，影响阅读体验。

## 变更内容
- 为学习笔记文件读取增加编码识别与回退解码能力
- 保持目录结构、路由和现有页面交互不变，仅修复内容读取结果
- 优先兼容现有 `study/` 目录中的中文 Markdown/代码笔记文件

## 影响
- 受影响的代码：`lib/study.ts` 及可能涉及的笔记内容展示链路
- 受影响的功能：学习笔记内容加载与展示
- 用户可见变化：原本乱码的笔记内容恢复为正常中文显示

## 示例数据
- 当前截图中 `学习笔记 / 计算机操作系统 / 笔记` 页面已出现乱码
- 本地样例文件：`study/21_计算机操作系统/笔记.md`
- 该文件原始字节示例包含 `b'\\xb2\\xd9\\xd7\\xf7...'`，可确认并非 UTF-8 文本
